Escolhemos o software Sublime Merge pra apresentar aqui hoje. 

Antes de iniciarmos o nosso tutorial, eu gostaria de fazer uma breve introdução do
programa, com algumas das principais vantagens e desvantagens. 

Esse é um programa git client gráfico, ou seja, uma ferramenta que permite o
desenvolvedor trabalhar com o controle de versão de modo visual. 

Ele se assemelha a execução do git no terminal, por exemplo, mas ao ivés de 
digitar os comandos, ele trabalha com palavras chaves e uma visualização
em barra lateral dos commits, que é bem orgânica. 

Por isso que aqui no slide, coloquei que ele tem uma interface ágil.

Foi desenvolvido pelos mesmos criadores do Sublime Text, que é um editor de código/texto
que está no mercado desde 2008, uns 14 anos aí. O Sublime Merge foi lançado em 2018, sendo
bem mais novinho, tendo 10 anos de diferença com o irmão mais velho (Sublime Text).

Um ponto negativo, pelo menos pra mim que sou estudante, é que, apesar de poder avaliar gratuitamente,
pra poder usar continuamente é preciso comprar a licensa do software. 
Então sempre que você estiver ali mexendo nas ferramentas vai aparecer um pop upzinho perguntando
passiva-agressivamente se, já que está gostando, por que não paga a licensa de uma vez?

Mas no geral, é uma ferramenta bacana com a proposta de ser mais rápida pra "commitar" principalmente
nos arquivos que estão em "stage", esperando ali um direcionamento né.

Enfim, é isso. 

No primeiro slide temos também um QR code que leva a página principal do site e uma curiosidade
inútil mas interessante é que, se tivesse no pc, o botão em destaque pra baixar é de acordo com
o sistema operacional da sua máquina. Então, se você estiver no Windos, já fica brilhando o 
botão de baixar pra versão do windows. 

Se não me engano, no caso do mobile, o botão em destaque é o do Linux. O Sublime Merge não tem
versão mobile (Android/Ios).

No segundo slide, é a primeira tela que aparece depois de você ter baixado o arquivo executável no
seu computador e ter instalado o programa na máquina. 

É bem simples, nem tem muita opção de personalização, então a gente até desconsiderou essa parte 
aqui no slide. 

Então beleza, estando nessa tela, o Sublime Merge te dá algumas opções para selecionar o seu reposi-
tório, sendo: criar um novo repositório, abrir um repositório já existente ou clonar um repositório
usando uma URL. 

No caso de clonar um repositório do git, se não me engano ele já faz a conexão automática e cria um remote. Então
se der um push em um arquivo, ele sobe pro Git. Eu testei só uma vez pra ver se dava certo, e funcionou.


No primeiro exemplo, criamos um repositório local que nomeamos de "Sublime Merge" (criatividade tava em falta),
e no canto superior direito dá pra ver o caminho desse repositório na minha máquina. O texto ali de UNREGISTERED é
porque eu não tenho a licensa do software. 

Para fazer a configuração de ambiente, você vai em "set user details", digita seu nome e e-mail, pra ficar a marcação com a sua assinatura de
qualquer modificação que você fizer. 

As opções de "aplicar para configuração local" ou "aplicar para repositório local somente", de uma forma
bem básica definem o nível que essas informações (usuário) vão ser guardadas e "invocadas" pros repositórios
que você está criando. 

O git config --local só pode ser usado em um repositório git, e esta configuração só fica salva naquele repositório que você tá criando.
Já o --global é a configuração que vale para todos os repositórios que você criar. 

Depois é só confirmar que a seta dentro da caixa marcada aqui em cima vai ficar verde.

E no slide seguinte a gente já consegue ver o meu nome e e-mail fixados.

Pra adicionar, a gente completa essa apresentação com um breve tutorial de como criar um branch no Sublime Merge.
Bem fácil, bem simples, com ideia de mostrar um pouco da interface e o painelzinho de comando:
--- Ctrl+Shift+P abre a barra de comandos ou clica na barra em cima escrito "master" (que é um branch padrão quando você cria um repositório direto no Sublime)
--- Para adicionar um branch, basta digitar create branch + apelido.
--- Para sair, basta digitar checkout e ir para outro branch existente.
